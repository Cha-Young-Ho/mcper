## 기획서 템플릿

```markdown
# [기능명] 기획

## 개요
[기능의 비즈니스 가치와 목표]

## 유저 스토리
As a [역할]
I want [기능]
So that [가치]

## 수용 기준
- [ ] 정상 케이스 1
- [ ] 정상 케이스 2
- [ ] 엣지 케이스 1
- [ ] 에러 케이스 1

## QA 시나리오

### 정상
1. 입력: ...
   예상 결과: ...
   실제 결과: ...

### 엣지
1. 입력: (빈 값)
   예상 결과: 검증 에러
   실제 결과: ...

### 에러
1. 입력: ... (DB 오류 시뮬레이션)
   예상 결과: 500 에러 + 로그
   실제 결과: ...

## 기술 아키텍처
[ERD, 시퀀스 다이어그램]

## 작업 분해
- [ ] 데이터 모델 설계
- [ ] 비즈니스 로직 구현
- [ ] API 엔드포인트
- [ ] 테스트 코드
- [ ] 문서 작성

## 리스크
- R1: [리스크명] → [대응]
- R2: [리스크명] → [대응]

## 예상 작업량
- 설계: 4시간
- 구현: 12시간
- 테스트: 8시간
- 문서: 2시간
- **총합**: 26시간
```

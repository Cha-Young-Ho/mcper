# docs/FRONTEND.md — 프론트엔드 관례\n\n---\n\n## 어드민 UI 구조\n\n### 기술 스택\n\n- **템플릿**: Jinja2 (FastAPI)\n- **스타일**: Bootstrap 5\n- **상호작용**: HTMX + vanilla JS\n- **상태 관리**: 서버 세션\n\n### 디렉터리 구조\n\n```\napp/templates/\n├── base.html                 # 레이아웃\n├── admin/\n│   ├── dashboard.html        # 대시보드\n│   ├── specs.html            # 기획서 관리\n│   ├── rules.html            # 규칙 관리\n│   ├── tools.html            # 도구 통계\n│   └── settings.html         # 설정 (패스워드 변경 등)\n└── static/\n    ├── css/\n    │   ├── bootstrap.min.css\n    │   └── admin.css\n    └── js/\n        ├── htmx.min.js\n        └── admin.js\n```\n\n---\n\n## 페이지별 설계\n\n### 1. 대시보드 (`/admin`)\n\n**목적**: MCP 호출 통계 + 빠른 액션\n\n**요소**:\n- 상단: 통계 카드 (기획서 수, 규칙 버전, MCP 호출)\n- 중간: 최근 활동 (기획서 업로드, 규칙 변경)\n- 하단: 시스템 상태 (DB, Redis, Celery)\n\n**갱신 간격**: 5초 (SSE 또는 polling)\n\n---\n\n### 2. 기획서 관리 (`/admin/specs`)\n\n**목적**: 기획서 CRUD\n\n**기능**:\n- 테이블: 기획서 목록 (검색, 정렬, 페이지네이션)\n- 모달: 신규 기획서 추가\n- 상세: 기획서 수정/삭제\n\n**검색**: HTMX POST + 서버 필터링\n\n---\n\n### 3. 규칙 관리 (`/admin/rules`)\n\n**목적**: 규칙 발행 + 버전 이력\n\n**구조** (탭):\n- Global: 전역 규칙\n- Repository: 저장소별 규칙\n- App: 앱별 규칙\n\n**각 탭**:\n- 카드: 현재 버전 (읽기)\n- 텍스트 에디터: 신규 본문 (쓰기)\n- 버튼: Publish (새 버전)\n- 이력: 최근 5개 버전 (버전 비교)\n\n---\n\n### 4. 도구 통계 (`/admin/tools`)\n\n**목적**: MCP 도구별 호출 통계\n\n**표시**:\n- 테이블: 도구명, 호출 수, 마지막 호출 시간\n- 차트: 시간별 호출 분포\n\n---\n\n### 5. 설정 (`/admin/settings`)\n\n**목적**: 어드민 설정\n\n**항목**:\n- 패스워드 변경\n- MCP Host 화이트리스트 (관리만, 자동 등록은 db)\n- 로그 수준\n\n---\n\n## UI 패턴\n\n### 폼 (HTMX)\n\n```html\n<form hx-post=\"/admin/specs\" hx-target=\"#response\">\n    <input type=\"text\" name=\"title\" required>\n    <textarea name=\"content\" required></textarea>\n    <button type=\"submit\">추가</button>\n</form>\n<div id=\"response\"></div>\n```\n\n### 모달\n\n```html\n<button hx-get=\"/admin/specs/new\" hx-target=\"#modal\" data-bs-toggle=\"modal\" data-bs-target=\"#modal\">신규</button>\n\n<div class=\"modal fade\" id=\"modal\">\n    <div class=\"modal-dialog\">\n        <div class=\"modal-content\">\n            <form hx-post=\"/admin/specs\" hx-on::after-request=\"if(event.detail.successful) htmx.closeModal()\">\n                <!-- 폼 내용 -->\n            </form>\n        </div>\n    </div>\n</div>\n```\n\n### 테이블 (서버 렌더링)\n\n```html\n<table class=\"table\">\n    <thead>\n        <tr>\n            <th hx-get=\"/admin/specs?sort=title\">제목</th>\n            <th hx-get=\"/admin/specs?sort=created_at\">작성일</th>\n            <th>액션</th>\n        </tr>\n    </thead>\n    <tbody hx-target=\"this\">\n        {% for spec in specs %}\n        <tr>\n            <td>{{ spec.title }}</td>\n            <td>{{ spec.created_at | date }}</td>\n            <td>\n                <button hx-get=\"/admin/specs/{{ spec.id }}/edit\">수정</button>\n                <button hx-delete=\"/admin/specs/{{ spec.id }}\">삭제</button>\n            </td>\n        </tr>\n        {% endfor %}\n    </tbody>\n</table>\n```\n\n---\n\n## CSS 가이드\n\n### 클래스 네이밍\n\n```\nBEM (Block Element Modifier)\n\n.admin__header              #

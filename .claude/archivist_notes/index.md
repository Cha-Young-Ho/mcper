# 기록관 메모 인덱스

대용량 파일 및 다중 파일 분석 결과를 저장하는 중앙 저장소.
다른 에이전트가 같은 정보를 필요로 할 때 이 메모를 참고하면 재읽기를 줄일 수 있음.

| - | - | - | - |

(첫 분석 메모는 여기 목록에 추가될 것)

---

## 메모 작성 가이드

각 메모는 다음 포맷을 따름:

```markdown
---
name: {{주제}}
last_updated: {{YYYY-MM-DD}}
files_covered: [파일1, 파일2, ...]
---

## 개요
(50-100 단어 요약)

## 핵심 구조
(3-5개 주요 클래스/함수)

## 데이터 흐름
(입력 → 처리 → 출력)

## 주의점
(운영 시 체크할 항목)

## 크로스 참조
(다른 메모와의 연결고리)
```

## 효율성 추적

- **초기 목표**: 팀 전체 컨텍스트 토큰 30% 절감
- **측정**: 월별 토큰 사용량 비교 (기록관 추가 전/후)
- **성공 기준**: 대용량 파일 중복 읽기 제거

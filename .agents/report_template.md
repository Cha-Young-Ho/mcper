# 작업 보고서 템플릿

**모든 에이전트가 큰 작업 완료 후 작성해야 하는 표준 보고서 형식.**

---

## 작성 기준

**다음 중 하나라도 해당하면 보고서 작성:**

- 파일 3개 이상 수정
- 로직 변경 (버그픽스 제외)
- 새 기능 추가
- 아키텍처 결정
- 의존성 변경
- 예상 시간 초과

**대상:** `docs/dev_log.md` 에 append

---

## 보고서 포맷 (간결)

```markdown
## [날짜]: [에이전트] [작업 제목]

### 작업 내용
- 주요 변경 3-5줄 요약
- 파일: [file1.py], [file2.py] (변경된 파일만)

### 판단 이유
- Why: 왜 이렇게 했는가? (요청사항 vs 기술적 판단)
- Risk: 예상되는 위험요소 (있으면만 기록)

### 결과
- ✅ 완료 / ⚠️ 부분 완료 / ❌ 보류

### 다음 단계 (필요시)
- 후속 작업 또는 의존성
```

## 실제 예시

### ❌ 나쁜 예 (너무 길고 상세)

```
## 2026-03-30: @coder JWT 토큰 검증 추가

기존 코드에서 JWT 토큰 만료 검증이 없었다...
(길게 설명)

변경 파일:
- app/auth/dependencies.py (line 45-67)
- app/auth/service.py (line 120-145)
...
```

### ✅ 좋은 예 (간결)

```
## 2026-03-30: @coder JWT 토큰 만료 검증 추가

**작업 내용:**
- JWT 디코딩 시 exp claim 검증 추가
- 만료 토큰: 401 Unauthorized 응답
- 파일: `app/auth/dependencies.py`, `app/auth/service.py`

**판단 이유:**
- Why: 보안 평가 CRITICAL 항목 (Admin 기본 패스워드와 함께)
- Risk: 기존 토큰 사용 중인 클라이언트 → 재로그인 필요

**결과:** ✅ 완료

**다음 단계:**
- @tester: 만료 토큰 E2E 테스트 작성
- @infra: 배포 전 토큰 갱신 정책 확인
```

| 에이전트 | 작성 시점 | 예시 |

| `@planner` | 기획서 완성 후 | "QA 시나리오 3가지, 더미 데이터 준비, MVP vs Phase 2 구분" |
| `@senior` | 설계 문서 완성 후 | "아키텍처: 새로 추가할 파일 3개, 의존성 변경 없음, 위험요소 없음" |
| `@coder` | 코드 작성 완료 후 | "파일 2개 수정, 함수 5개 추가, 기존 코드 영향도 없음" |

## 보고서 구조 상세

### 1. 작업 내용 (What)
```
- 3-5줄 핵심 요약
- "OOO 기능 추가", "XXX 리팩토링", "YYY 버그 수정"
- 변경된 파일 명시 (전체 경로 아니어도 됨)
```

### 2. 판단 이유 (Why)
```
**Why:** 왜 이렇게 결정했는가?
- 사용자 요청 기반? "기획서 QA 시나리오 3번 반영"
- 기술적 판단? "성능 최적화를 위해 인덱스 추가"
- 보안 이유? "보안 평가 CRITICAL 항목"

**Risk:** 예상 위험요소 (있으면만)
- "기존 API 호출자가 변경 필요할 수 있음"
- "DB 마이그레이션 필요"
```

### 3. 결과 (Status)
```
✅ 완료              (모든 작업 완료, 테스트 통과)
⚠️ 부분 완료          (주요 기능은 완료, 일부 남음)
❌ 보류              (의존성 또는 막힘)
🔄 진행 중           (아직 완료 안 됨, 중간 보고)
```

### 4. 다음 단계 (Then)
```
- 후속 에이전트: "@tester: E2E 테스트 작성 필요"
- 외부 의존성: "DB 마이그레이션 스크립트 필요"
- 사용자 확인: "배포 전 성능 테스트 필요?"
```

## 라이팅 팁

**간결함:**
- ❌ "저는 다음과 같은 이유로 이렇게 했습니다..."
- ✅ "Why: 보안 CRITICAL 항목이므로"

**명확함:**
- ❌ "여러 파일 수정했음"
- ✅ "파일: `app/auth/dependencies.py`, `app/services/auth_service.py`"

**실행성:**
- ❌ "나중에 테스트 필요할 수도"
- ✅ "@tester: 만료 토큰 E2E 테스트 추가 필요"

**분석:**
- ❌ "이 부분이 좋다고 생각함"
- ✅ "Why: 벡터 검색 실패 시 FTS 폴백으로 사용자 경험 개선"

## 저장 위치

```
docs/dev_log.md

## [날짜]: [에이전트] [제목]

### 작업 내용
...

### 판단 이유
...

### 결과
...
```

**누적 기록:**
- 같은 날 여러 작업 → 시간 표기 (예: "2026-03-30 14:30")
- 작업 완료 후 즉시 기록 (나중에 기억 못할 수 있음)
- dev_log 맨 위에 추가 (최신순)

## 체크리스트

보고서 작성 전 확인:

- [ ] 작업 제목이 명확한가? ("JWT 검증 추가" O, "기술 변경" X)
- [ ] 변경된 파일을 정확히 나열했는가?
- [ ] Why가 명확한가? (요청 vs 판단)
- [ ] Risk가 있으면 언급했는가?
- [ ] 다음 단계를 명시했는가?
- [ ] 100-300 단어 범위인가? (너무 길지 않은가?)
- [ ] 다른 에이전트가 읽을 수 있는가?

## 보고서 예시 모음

### 1. @planner 보고서

```
## 2026-03-30: @planner RAG 검색 기획서 작성

**작업 내용:**
- 사용자 스토리: 스펙 검색 3가지 시나리오
- QA 시나리오: 정상 + 벡터 실패 폴백 + 대용량 검색
- 더미 데이터: 스펙 10개, 청크 50개
- 우선순위: MVP (정상 경로) vs Phase 2 (고급 필터)

**판단 이유:**
- Why: 평가 보고서에서 "RAG 불완전" 지적, 엣지 케이스 명시화 필요

**결과:** ✅ 완료

**다음 단계:**
- @senior: 벡터 + FTS 하이브리드 아키텍처 설계
- @tester: QA 시나리오 테스트 케이스 추가
```

### 2. @coder 보고서

```
## 2026-03-30 14:30: @coder Admin 패스워드 강제 변경

**작업 내용:**
- app/main.py: 기동 시 기본 패스워드 감지 → 초기화 강제
- app/routers/admin.py: 패스워드 설정 엔드포인트 추가
- 파일: `app/main.py`, `app/routers/admin.py`

**판단 이유:**
- Why: 보안 평가 CRITICAL (Admin 기본 패스워드 미흡)
- Risk: 기존 자동 배포 스크립트가 기본값 사용하면 실패

**결과:** ✅ 완료

**다음 단계:**
- @tester: 기본 패스워드 로그인 거부 E2E 테스트
- @infra: 배포 스크립트 패스워드 설정 확인
```

### 3. @tester 보고서

```
## 2026-03-30 15:45: @tester RAG 검색 테스트

**작업 내용:**
- 단위 테스트: 벡터 검색 5개, FTS 폴백 3개
- 통합 테스트: pgvector + Elasticsearch 상호작용 2개
- 엣지 케이스: 빈 쿼리, 메모리 부족, 벡터 오류
- 커버리지: 85% (목표 70% 초과)

**판단 이유:**
- Why: RAG은 핵심 기능, CodeNode 파서 부재까지 포함한 광범위 테스트 필요
- Risk: 벡터 임베딩 모델 변경 시 모든 테스트 재실행

**결과:** ✅ 완료 (1개 엣지 케이스 버그 발견)

**다음 단계:**
- @coder: 벡터 오류 처리 로직 보완
- @infra: 메모리 부족 시나리오 부하 테스트
```

### 4. @infra 보고서

```
## 2026-03-30 16:30: @infra 배포 검수

**작업 내용:**
- 보안: Admin 패스워드 검증 확인, CORS 화이트리스트 추가
- 성능: Celery 작업 큐 깊이 모니터링 추천
- 배포: Docker 이미지 크기 2% 증가 (무시), 환경변수 신규 1개

**판단 이유:**
- Why: 보안 평가 3개 개선사항 + 성능 모니터링 추가
- Risk: 환경변수 변경 → 기존 배포 스크립트 업데이트 필요

**결과:** ✅ 배포 가능

**다음 단계:**
- 배포: 환경변수 가이드 문서 업데이트
```
